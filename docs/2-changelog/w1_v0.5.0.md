# Changelog - Week 1, 16-07-2026, V. 0.5.0

**Release Date**: Week 1, 16-07-2026 at 16:06

**Code review**: `docs/3-code-review/CR_w1_v0.5.0.md` (Codex loop, 2 rounds -> APPROVED)

## Changes

- **Feature**: `ColorPicker` (previously Add-page only) is now reusable on `DetailPage`, so any already-logged brew with a photo — including every seeded demo brew — can have its colour re-picked after the fact. A new `autoExtract` prop (default `true`, `false` on Detail) stops the picker's mount effect from auto-overwriting a brew's saved colour; the user must explicitly tap "Pick from photo" and sample a pixel to change it.
- **Sync**: `color` (`#rrggbb` hex, validated) joins the canonical `update` outbox patch — extended symmetrically across `normalizePatch`, `toPatchRow`, and `applyPatch` in `src/lib/sync.js`. No Supabase migration needed (the `entries.color` column already existed); no new op type or ordering change.
- **UI**: New "Drink colour" row on `DetailPage` with a display swatch and an edit state (swatch + "Pick from photo" + Save/Cancel), following the existing inline-edit pattern. Entries without a usable photo (`entry.photo` falsy) show a read-only swatch instead of an empty picker. New theme-variable-only styles in `src/index.css`.
- **Tests**: `src/lib/sync.test.js` gained `color` coverage — hex validation/rejection, `toPatchRow` mapping, and `applyPatch` round-trip.
- **Verification**: manual walkthrough confirmed the colour edit persists in-session (demo mode), resets on reload per the demo reseed gate (by design), and renders correctly across all five themes.
