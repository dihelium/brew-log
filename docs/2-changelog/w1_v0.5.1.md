## Changes

Fixes the two seeded demo coffee brews rendering with the same colour.

- **Root cause**: both demo coffees (Orange Spiced Cappuccino and Vanilla Latte) carried `#c97b3a`, which is the app's coffee accent fallback (`--accent-coffee`, `src/index.css:32`). Their source entries in `brew-log-backup-2026-06-20.json` had no custom colour, so both fell through to the default and looked identical.
- **Fix (`scripts/extract-demo-photos.mjs`)**: the demo-data generator now supports an optional per-brew `color` override in its `selections` list, applied as `color ?? source.color ?? null` so an explicit value wins over the backup. The Orange Spiced Cappuccino is now a warmer, more saturated orange (`#d9772b`) and the Vanilla Latte a lighter, softer beige (`#c99a6c`).
- **Regeneration (`src/lib/demoData.js`)**: regenerated from the backup rather than hand-edited (the file is marked "Do not edit by hand"). Only the two `color` lines changed; the base64 photo payloads are byte-identical.

No entry-schema, IndexedDB, sync-logic, component, or Supabase changes. The fix affects demo-mode seed data only.
