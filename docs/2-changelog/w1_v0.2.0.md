# Changelog - Week 1, 15-07-2026, V. 0.2.0

**Release Date**: Week 1, 15-07-2026 at 09:59
**Version**: 0.2.0 (previously 0.1.0)
**Object**: feat: editable brew fields (date/rating/notes/name/type), tea type, and calendar view
**Code review**: `docs/3-code-review/CR_w1_v0.2.0.md` (Codex loop, 1 round -> APPROVED)

## Changes

- **Editable brew fields**: date & time, rating (with a Clear affordance), notes, name, and type are now editable inline on DetailPage, following the existing tap-to-edit location pattern under a single `editingField` state. Name cannot be blanked; an invalid/empty datetime disables Save.
- **Generalized update path**: `updateEntry` now accepts a fixed patch set (`name`, `type`, `rating`, `notes`, `timestamp`, `location`) via a new `normalizePatch` plus widened, symmetric `toPatchRow`/`applyPatch` in `sync.js`. All target columns already exist in `toRow`, so there is **no entry-schema change, no IndexedDB version bump, and no Supabase migration**. Editing a date re-sorts the feed by descending timestamp (including offline).
- **Sync safety**: the outbox `update` op stays a whitelisted partial row update that never touches Storage/`photo_path`; the remotely-deleted-row case keeps the project's delete-wins outcome, now pinned by a regression test.
- **New "tea" type**: brew-type metadata centralized in a pure `src/utils/brewTypes.js` (`BREW_TYPES`, `brewTypeLabel`, `brewTypeEmoji`) driving AddPage, EntryCard, and DetailPage; `--accent-tea`/`--accent-tea-dark`/`--badge-tea-bg` and `data-type="tea"` rules added to all five themes. Legacy/null types render via a safe label fallback.
- **Calendar view** (`/calendar`, non-default): a month grid with a dot on each day that has brews, reached from a 🗓 button on the Feed heading. Selecting a day shows that day's brew list and its "daily cup" mug. Month chevrons re-point the selected day into the new month (clamped to month-end). Built on pure `src/utils/calendar.js` (`buildMonthMatrix`, `brewDaysInMonth`, `entriesForDay`, `dayKey`).
- **DailyCup reuse**: added an optional `historical` caption prop so the mug's "today" wording becomes date-neutral on the calendar without changing Feed behavior.
- **Helpers/tests**: new pure utils `brewTypes`, `calendar`, and `datetimeLocal` with dedicated Vitest suites; new `sync` cases cover the widened patch mapping, `normalizePatch`, and the delete-wins conflict. Lint clean, build clean, 31 affected tests passing.

## Migration

None. All edited fields map to columns that already existed on the Supabase `entries` table.
