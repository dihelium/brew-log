## Changes

Fixes a confirmed sync incident (an iPhone accumulated 17 brews that never reached Supabase) and hardens the sync path so it can't silently freeze again.

- **Resilient `flushOutbox`** (`src/lib/sync.js`): a failing outbox op no longer aborts the whole pass. Ops are processed in seq order; a failure blocks only *that entry's* later ops (via `blockedIds`, preserving add→update→delete causal order), while other entries keep flushing. Failed ops stay queued for the next pass. Returns a summary `{ ok, flushed, failed, cleanupFailed, error }`.
- **Idempotent photo upload**: the `add` path now uploads with `upsert:false` and treats a duplicate/409 ("already exists") result as success, so a partial failure (photo uploaded, row upsert failed) is safely retryable without requiring a Storage `UPDATE` policy — the likely original cause of the stranded entries.
- **Storage-error visibility**: `delete`'s `storage.remove()` failures are counted as non-blocking `cleanupFailed` (the row delete stays authoritative); `pullRemote` reports photo `download()` failures via a `photoFailed` soft signal.
- **Sync-error chip**: `BrewContext` tracks a `syncError` flag (set on flush/pull failure, cleared on a clean pass) and exposes `retrySync`; `FeedPage` shows a "⚠️ Sync failed — tap to retry" chip (`SyncErrorChip`) for real sessions when sync fails.
- **Forced account chooser**: Google sign-in now passes `prompt: 'select_account'`, so a device can no longer silently auto-select the wrong account.
- **Account visibility**: `BackupControls` shows "Signed in as \<email\>" (real sessions) beside the existing Sign out button, and its import handler now `await`s `importEntries` (was reporting "Restored undefined brews").

Six new `sync.js` tests cover resilience, per-entry ordering, idempotent-retry, cleanup, and download soft-failure. No entry-schema, IndexedDB-version, or Supabase-migration changes.
