## Changes

Added a **demo mode** so reviewers can explore the app without a Google sign-in.

- **Entry / exit**: an "Explore the demo — no sign-in needed" button on `LoginPage` enters a synthetic local session (`{ id: 'demo', isDemo: true }`); a "🧪 Demo mode · Sign in" chip (`DemoBanner`) on `FeedPage` exits back to login. The session is persisted under `localStorage['brewlog:demo']` so it survives a PWA reload. A real Google session always takes precedence and clears the demo flag.
- **Seeded brews**: on every demo entry the isolated `brew-log-demo` cache is cleared and reseeded with three brews — Orange Spiced Cappuccino (coffee), Vanilla Latte (coffee), Hot Matcha Latte (matcha) — re-dated to the current day so the daily-cup mug is always full and current. Photo data lives in a dynamically-imported `src/lib/demoData.js` chunk (generated by `scripts/extract-demo-photos.mjs`), keeping it out of the main bundle.
- **Sync safety**: `BrewContext.runSync` is a hard no-op in demo mode, so `flushOutbox`/`pullRemote` never run — no writes to Supabase and no remote-delete pruning of demo entries. Demo mutations stay in the isolated cache and are wiped on the next reseed. Add/edit/delete remain fully interactive locally.
- **Cold-offline handling**: if the demo photo chunk can't be fetched (installed PWA, offline, first-ever demo entry), `FeedPage` shows a retry message; retry reloads the document (a failed dynamic import is otherwise cached in the module map and can't recover in-page).
- **Supporting changes**: `cache.clear()` accessor; `src/lib/demo.js` pure seed builders (`demoTimestamps`, `buildDemoEntries`) + `seedDemoCache`; new unit tests (`demo.test.js`, `cache.clear()`); `docs/ARCHI.md` updated.

No entry-schema change, no IndexedDB version bump, no Supabase migration — fully backward compatible.
